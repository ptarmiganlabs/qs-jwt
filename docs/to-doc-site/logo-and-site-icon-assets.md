# Consistent Logo and Site Icon Assets

qs-jwt now uses a single approved logo design as the source for website icons and other exported image formats.

## What changed

- A single master SVG is used as the starting point for generated logo assets.
- Browser tab icons, app-style icons, and PNG logo exports are generated from that approved SVG.
- This ensures the same visual identity is used across documentation, shared links, and branded download surfaces.

## Why this matters

People who use qs-jwt should be able to recognize the tool immediately, whether they are reading the documentation, bookmarking the site, or viewing a shared link preview.

Using one approved source image reduces the risk of inconsistent branding and makes future visual updates easier to manage.

## User-facing impact

- The documentation site can use a consistent favicon and icon set.
- Shared pages can use matching branding for previews and browser surfaces.
- Future logo refinements can be rolled out consistently from one approved design.

## Breaking changes or migration

There are no breaking changes for qs-jwt users or administrators.
